# Purpose

The backlog contains future work that is valuable but not currently part of the active sprint.

Items in this document are ideas waiting to be prioritized.

The backlog is **not** a roadmap.

Features move from the backlog to the roadmap only after they have been evaluated and prioritized.

# Backlog Rules

A feature enters the roadmap only if it:

- Supports the product vision.
- Improves decision making.
- Fits the architecture.
- Can be maintained long-term.

Avoid adding features simply because they are interesting.

Focus on features that strengthen PolySignal's core value.

---

# Definition of Ready

Before work begins on a backlog item:

- The problem is clearly defined.
- The value is understood.
- The scope is reasonable.
- Dependencies are identified.
- Success criteria are documented.

Only then should the item move into an active sprint.
